# Task discussion T053: Game Designer: UX-ревью механики подгрупп и внутренних дверей

## [2026-02-27 08:32] boss (author)

### Task description

Оценить UX новой механики подгрупп (внутренние двери) для игрока-нематематика.

Через Agent Bridge (или анализ кода) оцени:

1. **Понятность концепции подгруппы (1-10)**:
   - Понимает ли игрок, что такое 'внутренняя дверь'?
   - Ясно ли что нужно выбрать подмножество ключей?
   - Ясно ли что подмножество должно быть 'замкнутым'?

2. **Прогрессия обучения (1-10)**:
   - Level 13: достаточно ли мягкое введение?
   - Level 14 → 16: растёт ли сложность плавно?
   - Не слишком ли резкий переход от Акта 1?

3. **Feedback при ошибке (1-10)**:
   - Если игрок выбрал НЕ подгруппу — понятно ли ПОЧЕМУ?
   - Сообщение 'не замкнуто' — что это значит для нематематика?
   - Предложения по улучшению обратной связи

4. **Aha-момент (1-10)**:
   - Когда игрок открывает первую дверь — есть ли радость?
   - Видит ли он связь: 'эти ключи работают вместе'?

5. **Перегрузка интерфейса (1-10)**:
   - Не слишком ли много элементов на экране (кристаллы + ключи + двери)?
   - Нужно ли упрощать UI?

DELIVERABLES:
- Подробный UX-отчёт по каждому уровню
- Конкретные рекомендации для улучшения
- Приоритизация: что исправить до релиза, что может подождать
