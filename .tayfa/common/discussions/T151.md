# Task discussion T151: Layer 2 — ключ применяет двухшаговую анимацию (A → B → проверка)

## [2026-03-03 17:30] boss (author)

### Task description

ПРОБЛЕМА (баг S018): На Layer 2 нажатие ключа в key_bar НЕ применяет перестановку. Кристаллы не двигаются. Это НЕПРАВИЛЬНО.

ПРАВИЛЬНАЯ МЕХАНИКА — ДВУХШАГОВАЯ АНИМАЦИЯ:

Когда на панели слева выбрано задание (ключ A) и игрок нажимает ⊕ на ключе B:

```
Состояние 1 (текущее)
    ↓ [применить ключ A — анимация ~0.5с]
Состояние 2 (промежуточное)
    ↓ [применить ключ B — анимация ~0.5с]
Состояние 3 (финальное)

Если Состояние 3 == Состояние 1 → A∘B = e → B = A⁻¹ → ПАРА НАЙДЕНА ✅
Если Состояние 3 ≠ Состояние 1 → A∘B ≠ e → B ≠ A⁻¹ → НЕПРАВИЛЬНО ❌
```

Игрок ВИЗУАЛЬНО ВИДИТ композицию A∘B:
1. Кристаллы переезжают по ключу A (видит действие ключа задания)
2. Потом по ключу B (видит действие выбранного ключа)
3. Если вернулись в исходное — B обратный к A!

Это наглядная демонстрация: A ∘ A⁻¹ = e (тождество).

FLOW при нажатии ⊕ на ключе B (Layer 2):
```
1. Запомнить текущее состояние (start_arrangement)
2. Применить ключ A (из активного задания) → анимация ~0.5с
3. По завершении — применить ключ B → анимация ~0.5с
4. По завершении — сравнить текущее с start_arrangement:
   a. Если совпадает → ПАРА НАЙДЕНА:
      - Заблокировать слот, зелёная анимация
      - Кристаллы уже на месте (вернулись!)
   b. Если НЕ совпадает → НЕПРАВИЛЬНО:
      - Красная вспышка
      - Вернуть кристаллы в start_arrangement (анимация обратно ~0.3с)
```

Если задание НЕ выбрано:
- Просто применить ключ B (как на Layer 1) — одна анимация
