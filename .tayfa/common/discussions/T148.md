# Task discussion T148: Layer 2 — игрок выбирает ключ для поиска пары

## [2026-03-03 16:21] boss (author)

### Task description

ПРОБЛЕМА: Сейчас на Layer 2 _active_slot фиксирован — первый неспаренный. Игрок не может выбрать, какому ключу искать пару. Приходится решать по порядку.

РЕШЕНИЕ: Два клика вместо одного:
1. Первый клик на ⊕ ключа → этот ключ ВЫБРАН как "ищу пару для него"
2. Второй клик на ⊕ другого ключа → попытка спарить

НОВЫЙ FLOW:
```
Состояние: _selected_key = -1 (ничего не выбрано)

Клик ⊕ на ключе A:
  if _selected_key == -1:
    # Первый клик — выбираем ключ A
    _selected_key = A
    Подсветить кнопку A в key_bar (яркая рамка)
    Подсветить слот ключа A в mirror_pairs_panel (active стиль)
  else:
    # Второй клик — пытаемся спарить _selected_key с B
    result = try_pair(_selected_key, B)
    if result.success:
      Заблокировать оба, зелёная анимация
      _selected_key = -1
    else:
      Красная вспышка
      _selected_key = -1  # сбросить выбор
```

АЛЬТЕРНАТИВА: клик на тот же ключ = сброс выбора.
Уже спаренные ключи не реагируют на клик.

ИЗМЕНЕНИЯ:
- layer_mode_controller.gd: on_key_tapped_layer2() — двухфазная логика
- key_bar.gd: добавить визуальное состояние "selected" для кнопки
- mirror_pairs_panel.gd: set_active_slot(idx) — позволить задавать активный слот извне

### Acceptance criteria

- [ ] Первый клик выбирает ключ (подсветка)
- [ ] Второй клик пытается спарить
- [ ] Верная пара → зелёный, оба заблокированы
- [ ] Неверная пара → красная вспышка, сброс выбора
- [ ] Клик на уже спаренный ключ → ничего
- [ ] Клик на тот же ключ = сброс выбора
- [ ] Панель зеркал подсвечивает выбранный слот
