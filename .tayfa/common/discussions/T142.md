# Task discussion T142: Цвет кластера = цвет ключа-генератора подгруппы

## [2026-03-03 15:44] boss (author)

### Task description

Изменить цвет обводки кластера на room_map: вместо одного цвета темы слоя (золотой/красный) — использовать цвет ключа из room_state.colors.

СЕЙЧАС: Все кластеры на Layer 3 — одинаковый золотой. Все на Layer 4 — одинаковый красный. Невозможно отличить разные подгруппы.

НУЖНО: Каждая подгруппа / пара / класс обводится СВОИМ цветом, взятым из room_state.colors:

Layer 2 (зеркальные пары):
- color = room_state.colors[key_room_idx] — цвет первого ключа пары

Layer 3 (подгруппы):
- color = room_state.colors[min_non_identity_element] — цвет наименьшего non-identity элемента подгруппы
- Пример: подгруппа {e, 2, 5} → color = room_state.colors[2]

Layer 4 (нормальные подгруппы):
- Аналогично Layer 3

Layer 5 (классы смежности):
- color = room_state.colors[representative_room_idx] — цвет представителя класса
- Пример: класс {1, 3, 4} с representative=1 → color = room_state.colors[1]

ФАЙЛ: Вызывающий код в layer_mode_controller.gd передаёт color в set_room_clusters().
room_map_panel.gd НЕ нужно менять — он уже принимает произвольный color.

### Acceptance criteria

- [ ] Разные подгруппы имеют РАЗНЫЙ цвет обводки
- [ ] Цвет берётся из room_state.colors, а не из фиксированной палитры слоя
- [ ] Layer 2: цвет = цвет ключа пары
- [ ] Layer 3: цвет = цвет генератора подгруппы
- [ ] Layer 4: цвет = цвет генератора
- [ ] Layer 5: цвет = цвет представителя класса смежности
